Lesson 1 · Sprites, Motion, Variables
Meet Your Shark 🦈
Stage setup · Arrow key movement · Score, Lives, Health variables · HUD display
🌊
▼
By the end of Lesson 1 you'll have an ocean scene, a shark that moves smoothly in all four directions, and a working HUD (heads-up display) showing Score, Lives, and Health on screen. You'll understand what variables are and why games need them.
Part 1 — Setting Up the Ocean
  • Open ScratchGo to scratch.mit.edu → Click Create. You'll see: the Stage (your game window, 480×360 pixels), the Sprite List, and the Code Blocks on the left.
  • Add your first backdrop — Shallow WaterClick the Stage thumbnail (bottom-right) → Choose a Backdrop → search "Underwater 1". This will be your Level 1 background. Later you'll add more backdrops for higher levels.
  • Delete the cat, add a SharkRight-click the cat → Delete. Click Choose a Sprite → search Shark. If you can't find one, use Paint to draw a simple shark shape. Resize it so it fits comfortably on the stage (try 80px wide).
  • Position the shark at the centre-leftDrag the shark to around x: -150, y: 0 on the stage. This will be its start position.
🧠 Key Concept: The Coordinate System

Scratch uses X and Y coordinates. The centre of the stage is (0, 0). X goes from -240 (left edge) to +240 (right edge). Y goes from -180 (bottom) to +180 (top). Every sprite has an X and Y position at all times.

Part 2 — Moving the Shark
  • Click on the Shark sprite — all code goes hereMake sure the shark is highlighted in the sprite list before dragging blocks.
  • Build the movement code:
    🚩 when Green Flag clicked
    go to x: (-150) y: (0)
    forever
    if <key [right arrow ▼] pressed?> then
    change x by (6)
    point in direction (90)
    if <key [left arrow ▼] pressed?> then
    change x by (-6)
    point in direction (-90)
    if <key [up arrow ▼] pressed?> then
    change y by (5)
    if <key [down arrow ▼] pressed?> then
    change y by (-5)
    Press the green flag and test it. The shark should move in all 4 directions.
  • Keep the shark inside the stageAdd this inside the forever loop to stop the shark going off-screen:
    if on edge, bounce
    Or for more control, use: if x position > 230 → set x to 230 (repeat for all 4 edges).
The forever loop runs 30 times per second. Each time it checks: "is a key being pressed?" This is called a polling loop — the same technique used in real game engines.
Part 3 — Creating All Your Variables

Now we'll create all the variables the game will need. Go to Variables → Make a Variable for each one. Set them all to "For all sprites".

Part 4 — The startGame Function (Preview)
  • Set all starting values when the green flag is clickedAdd these at the top of your shark's green flag script, above the forever loop:

    In Lesson 3 we'll move all of this into a proper named function called startGame.
Part 5 — HUD: Showing Variables on Screen
  • Show Score, Lives and Health on screenTick the checkbox next to each variable in the Variables palette. They appear as monitors on the stage. Drag them to neat positions — Score top-left, Lives top-centre, Health top-right.
  • Hide FishSpeed, PoisonActive, Level from the playerUntick those checkboxes — the player doesn't need to see those. Or right-click a monitor and choose hide.
In a real game you'd build a custom health bar sprite. For now the variable monitor is fine — we'll polish it in Lesson 6.

Release the Fish! 🐟
Clones · Random spawning · Collision detection · Multi-value fish
🐠
▼
By the end of Lesson 2 your game will have multiple types of fish spawning from the right side. Catching them adds to the score (different fish = different points). Missing them costs a life. A golden fish appears rarely for bonus points.
Part 1 — Add Three Fish Sprites
  • Add a regular fishClick Choose a Sprite → search Fish. Pick one and rename it "Fish" in the sprite info panel. Resize it to about 50px wide.
  • Add a second fish — the Bonus FishAdd another fish sprite. Rename it "GoldFish". Change its colour in the Costumes tab — select the fish shape and use the fill tool to make it golden/yellow. This fish will be worth 3 points.
  • Add a third fish — the Speed FishAdd a third fish sprite, rename it "FastFish". Make it a bright red/pink colour. This fish moves fast but is only worth 1 point.
🧠 Key Concept: Clones

A clone is a temporary copy of a sprite. The original sprite hides itself and acts as a factory — it produces clones over and over. Each clone runs its own code, moves independently, and can delete itself. This lets you have many fish on screen without needing many sprites.

Part 2 — Regular Fish Clone System
  • Green flag script — the factory
    🚩 when Green Flag clicked
    hide
    forever
    create clone of [myself ▼]
    wait (pick random (1) to (3)) seconds
  • Clone startup script — each fish behaves on its own
    when I start as a clone
    show
    go to x: (240) y: (pick random (-160) to (160))
    set size to (pick random (60) to (100))%
    forever
    change x by ((0) - (FishSpeed))
    if <touching [Shark ▼]?> then
    change [Score ▼] by (1)
    play sound [Pop ▼]
    delete this clone
    if <(x position) < (-240)> then
    change [Lives ▼] by (-1)
    delete this clone
Using pick random for both the Y position and size makes each fish feel unique. Varying size also adds an illusion of depth — bigger fish look closer.
Part 3 — Golden Fish (3 points, rare)
  • GoldFish spawns less often and is worth more
    🚩 when Green Flag clicked
    hide
    forever
    create clone of [myself ▼]
    wait (pick random (5) to (10)) seconds

    when I start as a clone
    show
    go to x: (240) y: (pick random (-150) to (150))
    forever
    change x by ((0) - (FishSpeed))
    change y by (pick random (-2) to (2))
    if <touching [Shark ▼]?> then
    change [Score ▼] by (3)
    play sound [Ding ▼]
    delete this clone
    if <(x position) < (-240)> then
    delete this clone
    Golden fish do NOT cost a life if missed — they're a bonus opportunity only. The wobbly Y movement makes them harder to catch.
Part 4 — Fast Fish (hard to catch, 1 point)
  • FastFish uses a local speed multiplier
    🚩 when Green Flag clicked
    hide
    forever
    create clone of [myself ▼]
    wait (pick random (3) to (6)) seconds

    when I start as a clone
    show
    go to x: (240) y: (pick random (-150) to (150))
    forever
    change x by ((0) - ((FishSpeed) * (2)))
    if <touching [Shark ▼]?> then
    change [Score ▼] by (1)
    delete this clone
    if <(x position) < (-240)> then
    change [Lives ▼] by (-1)
    delete this clone
    Multiplying FishSpeed by 2 means FastFish always moves at double the current level speed — this scales automatically as difficulty increases.
Part 5 — Game Over Check
  • On the Shark sprite, monitor lives in the forever loop
    if <(Lives) < (1)> then
    broadcast [GameOver ▼]
    stop [this script ▼]
    We'll build the proper Game Over screen in Lesson 4. For now this just stops the shark moving.
HiScore: Add this check after updating Score — if Score > HiScore → set HiScore to Score. The HiScore variable never gets reset by startGame, so it persists between games in the same session.

Lesson 3 · Custom Blocks (Functions)
Tidy Code with Functions 🧩
Define blocks · Parameters · Abstraction · DRY principle
⚙️
▼
By the end of Lesson 3 you'll have reorganised your entire game into clean, named custom blocks: startGame, catchFish(points), loseLife, damageShark(amount), spawnFish, and nextLevel. You'll understand why functions are one of the most important tools in programming.
🧠 What is a Function?

A function (called a Custom Block in Scratch) is a named chunk of code you can reuse. Instead of writing the same logic in ten places, you write it once, give it a name, and call that name whenever you need it.

The two big benefits: DRY (Don't Repeat Yourself — less code to write) and maintainability (if you want to change how catching a fish works, you change it in one place, not everywhere it happens).

Function 1 — startGame (Shark sprite)
  • Create the custom blockClick the Shark sprite → My Blocks → Make a Block → name it startGame → click OK.
  • Move all your setup code into it
    define startGame
    set [Score ▼] to (0)
    set [Lives ▼] to (3)
    set [Health ▼] to (100)
    set [Level ▼] to (1)
    set [FishSpeed ▼] to (3)
    set [TimeBonus ▼] to (30)
    set [PoisonActive ▼] to (0)
    go to x: (-150) y: (0)
    show
    switch backdrop to [Underwater 1 ▼]
  • Call it from the green flag
    🚩 when Green Flag clicked
    startGame
    forever
    — (movement code) —
Function 2 — catchFish(points) with a Parameter
  • Create catchFish with an inputGo to My Blocks (on the Fish sprite) → Make a Block → name it catchFish. Click Add an input (number or text) and name it points. Click OK.
  • Build the block using the parameter
    define catchFish (points)
    change [Score ▼] by (points)
    if <(Score) > (HiScore)> then
    set [HiScore ▼] to (Score)
    play sound [Pop ▼] until done
    delete this clone
  • Call it differently for each fish type
    catchFish (1)
    ← for regular fish
    catchFish (3)
    ← for golden fish
    catchFish (1)
    ← for fast fish
    Same function, different arguments. The word inside the brackets is called an argument — the value you pass into the parameter.
Function 3 — loseLife (Fish sprite)
  • Create loseLife on the Fish sprite
    define loseLife
    change [Lives ▼] by (-1)
    play sound [Oops ▼]
    if <(Lives) < (1)> then
    broadcast [GameOver ▼]
Function 4 — damageShark(amount) (Shark sprite)
  • This will be used by enemies in Lesson 4
    define damageShark (amount)
    if <(PoisonActive) = (0)> then
    change [Health ▼] by ((0) - (amount))
    set [color ▼] effect to (50)
    wait (0.2) seconds
    clear graphic effects
    if <(Health) < (1)> then
    change [Lives ▼] by (-1)
    set [Health ▼] to (100)
    if <(Lives) < (1)> then
    broadcast [GameOver ▼]
    When health hits zero, the shark loses a life but health resets to 100 — giving the player a chance to survive.
Function 5 — nextLevel (Shark sprite)
  • Increase difficulty and update the scene
    define nextLevel
    change [Level ▼] by (1)
    change [FishSpeed ▼] by (1)
    change [Score ▼] by (TimeBonus)
    set [Health ▼] to (100)
    set [TimeBonus ▼] to (30)
    next backdrop
    say (join [Level ] (Level)) for (2) secs
    broadcast [LevelUp ▼]
Notice how nextLevel also adds the TimeBonus to Score and resets Health — the player is rewarded for completing a level quickly and starts fresh. This is a game design decision, not just a coding choice.
Calling nextLevel — Every 5 Fish Caught
  • Add this check to the shark's forever loop
    if <((Score) mod (5)) = (0) and (Score) > (0)> then
    nextLevel
    mod (modulo) gives the remainder of division. 15 mod 5 = 0, 16 mod 5 = 1. So this fires at exactly 5, 10, 15, 20 fish caught.
🔥 Extension Challenge — Function with boolean parameter

Edit loseLife to accept a boolean parameter showWarning. If it's 1, show a red flash on screen. If it's 0, just deduct quietly. This teaches conditional logic inside functions.

✅ Lesson 3 Checklist
  • startGame resets all 8 variables and is called from the green flag
  • catchFish(points) uses a parameter and updates HiScore
  • loseLife deducts a life and broadcasts GameOver when Lives = 0
  • damageShark(amount) reduces Health and handles health = 0
  • nextLevel increases Level and FishSpeed, adds TimeBonus to Score, changes backdrop
  • The game still plays exactly as before — functions didn't change behaviour, just organisation
  • You can explain: what a parameter is, what DRY means, why functions help with debugging

Lesson 4 · Enemies, Health & Hazards
Danger in the Deep ☠️
Jellyfish · Mines · Pufferfish · Octopus · Health system · Poison effect
🪼
▼
By the end of Lesson 4 your shark will face five types of hazard: a jellyfish that drains health, a mine that causes instant life loss, a pufferfish that poisons the shark slowing it down, an octopus that blocks movement, and a whirlpool that briefly pulls the shark off course.
☠️ Enemies & Hazards Guide
🪼 Jellyfish
Floats up and down slowly. Appears from Level 1.
Effect: -10 Health per second of contact
💣 Mine
Drifts slowly across the screen. Appears from Level 2.
Effect: -1 Life instantly on touch
🐡 Pufferfish
Swims like a fish but inflates when touched. Level 3+.
Effect: Activates PoisonActive for 5 seconds — shark moves at half speed
🐙 Octopus
Emerges from the bottom. Tentacles grab the shark. Level 4+.
Effect: Freezes shark movement for 2 seconds, -15 Health
🌀 Whirlpool
Appears randomly on screen, slowly rotates. Level 5+.
Effect: Pushes shark in a random direction, -5 Health
Hazard 1 — Jellyfish
  • Add a Jellyfish spriteSearch the sprite library for "Jellyfish" or draw one. Rename it "Jellyfish". Make it semi-transparent in the costumes editor (set fill opacity to 70%).
  • Factory script (green flag)
    🚩 when Green Flag clicked
    hide
    forever
    create clone of [myself ▼]
    wait (pick random (4) to (8)) seconds
  • Clone script — floating movement
    when I start as a clone
    show
    go to x: (pick random (-200) to (200)) y: (-200)
    forever
    change y by (2)
    change x by (pick random (-1) to (1))
    if <touching [Shark ▼]?> then
    damageShark (10)
    wait (0.5) seconds
    if <(y position) > (200)> then
    delete this clone
    The wait 0.5 seconds after dealing damage prevents the shark from losing all health instantly — it caps the damage rate to 20 HP per second.
Hazard 2 — Mine
  • Add a Mine spriteDraw a simple black circle with small spikes (like a sea mine) in the Paint editor, or search the library. Rename it "Mine".
  • Mine clone script

    hide
    forever
    if <(Level) > (1)> then
    create clone of [myself ▼]
    wait (pick random (6) to (12)) seconds

    when I start as a clone
    show
    go to x: (240) y: (pick random (-150) to (150))
    forever
    change x by (-1)
    if <touching [Shark ▼]?> then
    change [Lives ▼] by (-1)
    set [ghost ▼] effect to (80)
    wait (0.3) seconds
    delete this clone
    if <(x position) < (-250)> then
    delete this clone
    Mines only start appearing from Level 2 (the if Level > 1 check). They're slow but deadly — instant life loss.
Hazard 3 — Pufferfish (Poison)
  • Add a Pufferfish spriteSearch the library or paint a round fish. Give it two costumes: normal (small) and puffed up (large circle). Rename it "Pufferfish".
  • Poison effect script
    when I start as a clone
    show
    switch costume to [puffer-normal ▼]
    go to x: (240) y: (pick random (-150) to (150))
    forever
    change x by ((0) - (FishSpeed))
    if <touching [Shark ▼]?> then
    switch costume to [puffer-big ▼]
    set [PoisonActive ▼] to (1)
    wait (5) seconds
    set [PoisonActive ▼] to (0)
    delete this clone
  • Slow the shark when poisoned — modify movement code In the shark's movement forever loop, change the movement amounts:
    if <key [right arrow ▼] pressed?> then
    if <(PoisonActive) = (1)> then
    change x by (3)
    else
    change x by (6)
    Repeat this pattern for all 4 directions. When poisoned, the shark moves at half speed. The same pattern goes for Y movement.
Hazard 4 — Octopus (Freeze)
  • Add Octopus sprite from the libraryRename it "Octopus". It will rise from the bottom and broadcast a freeze event when touched.
  • Octopus clone + freeze broadcast
    when I start as a clone
    show
    go to x: (pick random (-180) to (180)) y: (-200)
    repeat (60)
    change y by (3)
    if <touching [Shark ▼]?> then
    broadcast [FreezeShark ▼]
    damageShark (15)
    delete this clone
    delete this clone

    when I receive [FreezeShark ▼]
    ← Add this to the SHARK sprite
    stop [other scripts in sprite ▼]

Using broadcast to communicate between sprites is a key Scratch technique. It's a simplified version of the event-driven programming pattern used in real apps and games.
Hazard 5 — Whirlpool

  • Whirlpool script — push effect

    show
    go to x: (pick random (-180) to (180)) y: (pick random (-150) to (150))
    repeat (200)
    next costume
    wait (0.1) seconds
    if <touching [Shark ▼]?> then
    change x by (pick random (-20) to (20))
    change y by (pick random (-20) to (20))
    damageShark (5)
    delete this clone

    🚩 when Green Flag clicked
    forever
    set size to (Health) %
    if <(Health) > (60)> then
    set [color ▼] effect to (0)
    else if <(Health) > (30)> then
    set [color ▼] effect to (25)
    else
    set [color ▼] effect to (0) ... and make it flash red
    Green = healthy (60–100), yellow = caution (30–60), red and flashing = danger (0–30). This gives the player at-a-glance health feedback.
